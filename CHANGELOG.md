### Added

- **OBS-004**: public `GET /v1/quality` endpoint returning the latest live Ragas eval run `{"faithfulness","answer_relevancy","context_precision","context_recall","golden_set_version","finished_at"}` (6 keys exact, `NUMERIC(5,4)` decoded as JSON number without rounding via `bigdecimal` crate). No live row → `200 {"status":"no_data"}`. DB unavailable → 503 sanitised `{"error":"upstream_unavailable","request_id":"<uuid>"}`. No auth gate (public anonymous). `#quality-widget` added to `/observability` (`observability.html/js/css`) with independent `loadQuality()` fetch (failure does not affect `#usage-widget`/`#health-widget`). No migration, no OpenAPI change. 5 security headers preserved. Fix: `bigdecimal` feature changed from `serde` to `serde-json`; metric fields annotated `#[serde(with = "bigdecimal::serde::json_num")]` to emit JSON numbers (not strings) as required by AC-1/AC-3; regression test `serialization_bigdecimal_emits_json_number_not_string` added (no-DB, runs under plain `cargo test`). Fix (review MED): `GET /v1/quality`, `GET /v1/stats`, `GET /v1/status` 200 paths now emit `Content-Type: application/json; charset=utf-8` (security.md §A05) via shared `handlers::json_utf8` helper; exactly one `content-type` header produced (header array `insert` replaces `Json`'s own value).

- **OBS-003**: `--persist` flag for `eval/ragas_runner.py` — when `--mode live --persist` is provided, the runner inserts one aggregated row per run into the new `eval_runs` table (migration `0005_eval_runs.sql`). Columns: 4 Ragas metrics (`NUMERIC(5,4) NOT NULL`), run metadata (`git_sha`, `started_at`, `finished_at`, `entries_total/ok/errors`), and `golden_set_version` (SHA-256 of `specs/golden_qa.jsonl` bytes). Offline runs are never persisted (`event=persist_skipped reason=offline_mode`). DB failures emit `event=persist_failed` (no DSN/stack leaked) and exit code `4`. Artifact-then-persist ordering guarantees the JSON run-file is intact on any DB failure. Zero LLM payload stored; INSERT is always parameterised (no f-string into SQL). Partial index `eval_runs_live_latest_idx` shapes the OBS-004 read path.

- **OBS-002**: public `GET /v1/status` endpoint returning aggregate service health `{"status":"ok"|"degraded","dependencies":{"postgres":{..},"gcs":{..},"workers":{..}},"checked_at":"<rfc3339>"}`. 3 probes run in parallel (AC-9), 2 s timeout each (AC-8), result cached in-process 10 s TTL (AC-10). 200-always (AC-5/AC-6), no auth gate (AC-7), sanitised body — no host/message/SQL detail (AC-2/AC-11). `#health-widget` added to `/observability` (`observability.html/js/css`). No migration, no OpenAPI change (AC-16), 5 security headers preserved (AC-15).
- **OBS-001**: public `/observability` page with `Chat | Observabilité` nav and a usage widget fed by new public `GET /v1/stats` endpoint returning exact `count(*)` of conversations (`{"conversation_count": <int>}`). DB unavailable → 503 sanitized `{"error":"upstream_unavailable","request_id":"<uuid>"}`. Nav added to `index.html`. New static assets `observability.{html,js,css}`. All 5 security headers preserved router-wide. No auth gate, no migration, no OpenAPI change.
