### (b) Three key hypotheses

1. **sqlx 0.8 has NO per-connect password callback on `PgConnectOptions`**, but `Pool::set_connect_options(...)` atomically swaps the options used for FUTURE physical connects. Resolution of OQ-1: **(i)** at boot, the boot path fetches the first IAM token and builds `PgConnectOptions::from_str(database_url)?.password(token.expose_secret())`, then calls `PgPoolOptions::new().max_connections(5).max_lifetime(45 min).before_acquire(<gatekeeper>).connect_with(opts).await?`. **(ii)** `before_acquire(|conn, meta| async move { reject if metadata.age + REFRESH_AHEAD > 3600s })` rejects connections whose underlying token is in the refresh-ahead window — sqlx then closes that connection and opens a new one. **(iii)** a single background task `tokio::spawn` wakes every 30 min, calls `sql_token_provider.get_or_refresh()`, builds a fresh `PgConnectOptions` with the new password, calls `pool.set_connect_options(...)`. Now NEW physical connects use the fresh token; combined with `max_lifetime = 45 min`, no physical connection ever survives past token expiry. This is the canonical sqlx pattern (no fork, no upstream issue). The `before_acquire` is a **defence-in-depth gate** (catches clock skew); the background refresh is the **nominal mechanism**. AC-4 wording "after_connect hook nominal" is corrected to "background refresh + set_connect_options + before_acquire gate" — flagged in Risks for human ack.
2. **asyncpg 0.30 `password=` accepts an async callable** (since 0.21.0). Resolution of OQ-2: `create_pool(..., password=lambda: token_provider.get_token())` where `get_token` is async. Each new physical connect invokes the callable, fetches a fresh token from the cache (refresh-ahead inside `SqlTokenProvider`). `max_inactive_connection_lifetime=45*60` complements `max_lifetime` to recycle idle connections before token expiry. No `connection_class` override, no pre-fetch hack. Spec AC-6 wording "à confirmer au plan" resolved → callable path retained.
3. **TokenProvider relocation is mechanically safe**. `gateway/src/gcs/token.rs` → `gateway/src/auth_metadata/token.rs`; `gateway/src/auth_metadata/mod.rs` (new, ~10 LOC) `pub mod token; pub use token::TokenProvider;`; `gateway/src/gcs/mod.rs` keeps `pub use crate::auth_metadata::token::TokenProvider;` re-export to preserve the `gcs::token::TokenProvider` import path used by `test_signed_url.rs:13` and `test_dashboard_backend.rs`. **Tests do not break.** `lib.rs` adds `pub mod auth_metadata;`.

## Goal

Resolve the INFRA-002 PR-f deploy blocker (Cloud Run IAM auth handshake rejected by Cloud SQL) by making gateway sqlx and workers asyncpg pools inject a fresh Cloud SQL IAM access token (scope `sqlservice.admin`) on every new physical connection, fetched from the Cloud Run metadata server with refresh-ahead caching. Relocates `TokenProvider` (SEC-004) to a shared `auth_metadata` module so the same struct backs two scope-distinct instances: `gcs_token_provider` (default scope, signBlob) and `sql_token_provider` (sqlservice.admin scope, Cloud SQL).

  ```python
  # SqlTokenProvider — Cloud SQL IAM access token from metadata server.
  # Dedicated httpx.AsyncClient (timeout 5.0, connect 2.0 — AC-7).
  # Cache + refresh-ahead (now >= expires_at - 60s) + asyncio.Lock.
  # bearer wrapped in pydantic.SecretStr (security.md §A09).

  METADATA_BASE_URL_DEFAULT = "http://metadata.google.internal"
  METADATA_TOKEN_PATH = "/computeMetadata/v1/instance/service-accounts/default/token"
  CLOUD_SQL_SCOPE = "https://www.googleapis.com/auth/sqlservice.admin"
  REFRESH_AHEAD_SECONDS = 60

  @dataclass
  class _CachedToken:
      bearer: SecretStr
      expires_at: datetime  # UTC

  class SqlTokenProvider:
      def __init__(self, *, base_url: str = METADATA_BASE_URL_DEFAULT,
                   scope: str = CLOUD_SQL_SCOPE) -> None:
          self._base_url = base_url
          self._scope = scope
          self._client = httpx.AsyncClient(timeout=httpx.Timeout(5.0, connect=2.0))
          self._lock = asyncio.Lock()
          self._cached: _CachedToken | None = None

      async def aclose(self) -> None:
          await self._client.aclose()

      async def get_or_refresh(self) -> SecretStr:
          if self._cached and datetime.now(UTC) < self._cached.expires_at - timedelta(seconds=REFRESH_AHEAD_SECONDS):
              return self._cached.bearer
          async with self._lock:
              if self._cached and datetime.now(UTC) < self._cached.expires_at - timedelta(seconds=REFRESH_AHEAD_SECONDS):
                  return self._cached.bearer
              self._cached = await self._fetch()
              return self._cached.bearer

      async def _fetch(self) -> _CachedToken:
          url = f"{self._base_url}{METADATA_TOKEN_PATH}"
          params = {"scopes": self._scope} if self._scope else None
          try:
              r = await self._client.get(url, headers={"Metadata-Flavor": "Google"}, params=params)
          except httpx.TimeoutException as e:
              raise TokenFetchError("timeout") from e
          except httpx.HTTPError as e:
              raise TokenFetchError("network") from e
          if r.status_code >= 400:
              raise TokenFetchError("metadata_token_failed")
          body = r.json()
          return _CachedToken(
              bearer=SecretStr(body["access_token"]),
              expires_at=datetime.now(UTC) + timedelta(seconds=int(body["expires_in"])),
          )
  ```

pub struct OAuthScope(&'static str);

### gateway `run()` skeleton (after refactor)

```rust
pub async fn run(config: Config) -> Result<()> {
    let sql_token_provider = Arc::new(TokenProvider::for_cloud_sql()?);
    let (initial_token, _) = sql_token_provider
        .get_or_refresh()
        .await
        .context("boot.sql_pool_init_failed reason_code=metadata_token_failed")?;
    let opts = sqlx::postgres::PgConnectOptions::from_str(&config.database_url)?
        .password(initial_token.expose_secret());
    let pool = sqlx::postgres::PgPoolOptions::new()
        .max_connections(5)
        .max_lifetime(Duration::from_secs(SQL_CONNECTION_MAX_LIFETIME_SECS))
        .before_acquire(|_conn, meta| Box::pin(async move {
            // AC-4 gate: reject if the underlying token is within 60 s of TTL.
            Ok(meta.age < Duration::from_secs(SQL_CONNECTION_MAX_LIFETIME_SECS - 60))
        }))
        .connect_with(opts)
        .await
        .context("boot.sql_pool_init_failed reason_code=cloud_sql_auth_failed")?;
    // Background refresher (every 30 min) — set_connect_options for future connects.
    spawn_sql_token_refresher(pool.clone(), sql_token_provider.clone(), config.database_url.clone());
    let state = Arc::new(AppState::new_with_pool_and_sql_token_provider(config, pool, sql_token_provider)?);
    // ... unchanged
}
```

## Risks / open questions

- **OQ-1 RESOLVED with caveat on AC-4 wording**: sqlx 0.8 has no per-connect password callback; the pattern is `Pool::set_connect_options(opts)` (atomic swap for future connects) + `before_acquire` (age gate) + background `tokio::spawn` refresher every 30 min. `max_lifetime = 45 min` ensures no physical connection outlives its token. AC-4 spec says "after_connect hook nominal" — this is technically inaccurate for sqlx 0.8. **Plan retains the contractual intent** ("fresh token per new physical connect, cache hit ≤ 1 ms per acquire") via the documented mechanism. **Awaits human ack** that this realization satisfies AC-4 (no spec amendment requested — phrasing reflects what the underlying library exposes). Citations: `docs.rs/sqlx/0.8/sqlx/struct.Pool.html#method.set_connect_options`, `docs.rs/sqlx/0.8/sqlx/pool/struct.PoolOptions.html`.
- **OQ-2 RESOLVED**: asyncpg ≥ 0.21.0 accepts a sync or async callable as `password=`, invoked per new connect. Workers pin `>= 0.30` → supported. Path retained: `password=<async lambda calling sql_token_provider.get_or_refresh()>`. Citation: `magicstack.github.io/asyncpg/current/api/index.html` `connect()` reference + 0.21.0 changelog.
- **OQ-5 SPLIT RECOMMENDED**: estimated total = 395 LOC excl. tests / 476 incl. tests. **Split into PR-A (gateway, ~210 LOC) + PR-B (workers, ~266 LOC)**. Gateway-first per `docs/blockers.md` 2026-05-29 (gateway boot is THE blocker; workers haven't reached prod yet). If human elects single PR despite the budget overrun, plan executes in same order with one branch.
- **AC-11(a) docker-compose Postgres password mismatch**: real Cloud SQL IAM injects a bearer token as PG password; docker-compose Postgres uses literal password `postgres`. The integration test cannot exercise BOTH layers simultaneously. Plan resolution: the test exercises (i) metadata-mock end (assert `.expect(1)` with `?scopes=sqlservice.admin`) and (ii) pool layer (`PgPoolOptions` + `before_acquire` against literal-password local Postgres). The IAM password wiring is tested at the unit level (token.rs) + asserted via mockito query string. **Trade-off acknowledged**: no end-to-end IAM-against-real-Cloud-SQL test phase 1 (matches spec Non-goals "Pas de live test --ignored").
- **CI Postgres for gateway job**: AC-11 requires Postgres in CI for `cargo test`. Today the `gateway` job has none. Plan adds `services: postgres:` to `.github/workflows/ci.yml` `gateway` job, mirroring `contract` job. **Side-effect**: all gateway tests post-merge run against Postgres — most don't need it. No breakage expected (sqlx tests already handle absent pool gracefully).
- **Workers `db_pool` conftest skip-vs-fail**: existing fixture skips if Postgres unreachable; AC-12 forbids CI skip for SEC-005 tests. Plan adds `db_pool_required` no-skip variant. **Does not modify existing `db_pool`** (avoids churn on RET-001 / ING-003 tests).
- **`secrecy 0.10` SecretString API**: `SecretString::from(String)` (current usage in token.rs L169) is correct. `pydantic.SecretStr` Python equivalent — both redact in repr/log per `security.md §A09`.
- **No new prod dep**: gateway reuses sqlx/reqwest/secrecy/chrono; workers reuses asyncpg/httpx; only dev-dep `respx` added to workers. No ADR needed (matches rule).
- **Background refresher fault tolerance**: if the 30-min refresh fails (metadata transient), the spawned task logs and continues; the pool keeps using the prior `connect_options` until the next tick. Combined with `max_lifetime=45 min`, a connection established just before a failed refresh will be recycled within 45 min — within token TTL of 60 min. **Acceptable** (matches failure mode 5 in spec).
