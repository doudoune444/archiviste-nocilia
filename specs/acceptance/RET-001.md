# RET-001 — Endpoint workers `POST /v1/retrieve` (top-K cosine, sans rerank)

## Contexte

ING-001 alimente la table `chunks` avec embeddings bge-m3 (vector(1024)) indexés HNSW cosine (FOUND-003 AC-5), mais aucun chemin de lecture n'existe : GEN-001 ne peut pas démarrer sans interface de retrieval. Ce ticket livre l'endpoint workers interne `POST /v1/retrieve` qui prend une `query` texte + `top_k` + `user_tier`, embed la query, exécute la recherche cosine pgvector filtrée ACL, et retourne les chunks ordonnés par score. Phase 1 sans rerank, sans hybrid search, sans cache : baseline minimale exploitable par le pipeline GEN.

## Critères d'acceptation

- AC-1 : Le worker FastAPI expose `POST /v1/retrieve` accessible sur le port workers (8000) ; un appel valide retourne `200` avec `Content-Type: application/json; charset=utf-8`.
- AC-2 : Le body de requête est un JSON de schéma `{"query": str, "top_k": int|absent, "user_tier": str}` ; tout autre champ supplémentaire est ignoré silencieusement, toute clé requise manquante retourne `400 {"error":"invalid_request"}`.
- AC-3 : `query` doit être une chaîne non-vide après strip et de taille ≤ 4096 octets UTF-8 ; sinon `400 {"error":"invalid_query"}`.
- AC-4 : `top_k` est un entier optionnel ; absent ou `null` → valeur par défaut `5` ; présent et hors `[1, 20]` (bornes incluses) → `400 {"error":"invalid_top_k"}`.
- AC-5 : `user_tier` est obligatoire et doit appartenir exactement à l'ensemble `{"anonymous","members","author_only"}` ; absent ou hors ensemble → `400 {"error":"invalid_user_tier"}`.
- AC-6 : La query est embeddée via le wrapper bge-m3 réutilisé de ING-001 (modèle `BAAI/bge-m3`, dimension 1024) ; le modèle est chargé une fois au démarrage du worker (lifespan startup), aucun chargement n'a lieu pendant le traitement d'une requête.
- AC-7 : La recherche exécute une requête SQL paramétrée unique utilisant l'opérateur pgvector `<=>` (cosine distance) sur `chunks.embedding`, joignant `documents` pour appliquer le filtre ACL post-retrieval `WHERE documents.access_tier = ANY($allowed_tiers)` et `LIMIT $top_k`, où `$allowed_tiers` est dérivé de `user_tier` selon la table : `anonymous → {public}`, `members → {public, members}`, `author_only → {public, members, author_only}`.
- AC-8 : Le filtre ACL est exprimé dans la clause SQL `WHERE` (pas en post-filtrage applicatif) ; un test d'intégration insérant 1 chunk `public` + 1 `members` + 1 `author_only` puis appelant `user_tier="anonymous"` retourne uniquement le chunk `public`, jamais les autres, indépendamment des scores cosine.
- AC-9 : La réponse `200` a la forme exacte `{"chunks":[{"chunk_id":str,"document_id":str,"source_path":str,"ord":int,"text":str,"score":float}, ...], "embedding_ms":int, "search_ms":int}` ; `chunk_id` et `document_id` sont des UUID au format canonique RFC 4122 ; `source_path` provient de `documents.source_path`.
- AC-10 : `score` est calculé comme `1 - (chunks.embedding <=> $query_embedding)` (similarité cosine ∈ `[0.0, 1.0]`) et arrondi à 6 décimales ; le tableau `chunks` est ordonné par `score` décroissant (égalités résolues par `chunk_id` ASC pour déterminisme).
- AC-11 : `len(chunks) ≤ top_k` toujours ; aucun doublon `chunk_id` dans la réponse.
- AC-12 : Si la recherche ne renvoie aucune ligne (table vide, ou aucun chunk éligible ACL), la réponse est `200 {"chunks":[], "embedding_ms":N, "search_ms":M}` — jamais `404` ni `204`.
- AC-13 : Si l'embedder n'est pas chargé (état startup non terminé ou erreur de chargement initial), la réponse est `503 {"error":"embedder_unavailable"}` et la requête n'atteint pas la DB.
- AC-14 : Si la connexion PostgreSQL échoue (timeout, refus, erreur transport) pendant la recherche, la réponse est `503 {"error":"database_unavailable"}` ; aucune trace de la query SQL ni du message d'erreur DB ne fuit dans le body de réponse.
- AC-15 : `embedding_ms` est la durée mesurée du seul appel d'embedding de la query (entrée fonction → vecteur retourné) en millisecondes entières ; `search_ms` est la durée mesurée du seul aller-retour SQL `SELECT ... LIMIT top_k` en millisecondes entières ; les deux sont `>= 0`.
- AC-16 : Pour chaque requête traitée (succès ou erreur métier), un log JSON unique est émis à la forme `{"event":"retrieve","query_len":N,"top_k":K,"user_tier":T,"results":R,"embedding_ms":E,"search_ms":S,"status":"ok|invalid_query|invalid_top_k|invalid_user_tier|embedder_unavailable|database_unavailable"}` ; `query_len` est la longueur en octets UTF-8 ; ni `query` (texte brut), ni `text` retourné, ni `embedding`, ni message d'erreur DB ne sont loggés.
- AC-17 : Le contrat REST `POST /v1/retrieve` (request schema, response schema, codes d'erreur) est ajouté à `specs/openapi/gateway-to-workers.yml` dans la même PR ; `schemathesis run specs/openapi/gateway-to-workers.yml` passe contre le worker démarré.

## Non-goals

- Pas de rerank (cross-encoder, RRF, MMR) — ticket RET-* futur.
- Pas de hybrid search (BM25 + vector) — futur.
- Pas de query expansion / HyDE / multi-query — futur.
- Pas de cache résultat (Redis ou autre) — ticket SEC-* / RET-* futur.
- Pas de filtre custom (tags, source_path glob, date range) en phase 1.
- Pas de pagination (cursor, offset) — `top_k` cap à 20 suffit.
- Pas d'authentification / vérification du `user_tier` côté worker — `user_tier` est un input de confiance fourni par la gateway (frontière de confiance W-S-1 traitée séparément en SEC-*).
- Pas de rate-limit côté worker — responsabilité gateway (G-D-1).
- Pas d'instrumentation OpenTelemetry / Langfuse — tickets OBS-* dédiés ; les durées dans le body de réponse suffisent au debugging local phase 1.
- Pas d'exposition publique : l'endpoint est interne (Cloud Run `ingress=internal` une fois déployé) et n'est pas atteint directement par l'extérieur.
- Pas de gestion `text` HTML-escape / sanitization — c'est la responsabilité du caller GEN-* qui consommera ces chunks dans un prompt LLM.

## Failure modes

- `query` vide / blanc-only / > 4096 octets → `400 {"error":"invalid_query"}`, log avec `status:"invalid_query"`, aucune écriture, aucun appel embedder.
- `top_k` non-int / < 1 / > 20 → `400 {"error":"invalid_top_k"}`, log avec `status:"invalid_top_k"`.
- `user_tier` absent / hors ensemble → `400 {"error":"invalid_user_tier"}`, log avec `status:"invalid_user_tier"`.
- Body JSON malformé / Content-Type non-JSON / champ `query` ou `user_tier` manquant → `400 {"error":"invalid_request"}`.
- Embedder non chargé au moment de la requête (startup pas terminé, ou échec de chargement initial) → `503 {"error":"embedder_unavailable"}`, log avec `status:"embedder_unavailable"`.
- Connexion DB indisponible / timeout SQL > 5s → `503 {"error":"database_unavailable"}`, log avec `status:"database_unavailable"` ; détail erreur DB jamais retourné au client.
- Zéro résultat (table vide ou aucun chunk éligible ACL) → `200 {"chunks":[], "embedding_ms":..., "search_ms":...}` (succès, pas une erreur).

## Performance / SLO

- p95 latence end-to-end < 500 ms sur corpus de 10 000 chunks (embed query bge-m3 CPU + search HNSW cosine), mesurée localement avec hardware baseline FOUND-002 (4 cœurs / 8 GiB RAM / SSD).
- Embedder préchargé au boot du worker : aucun cold-start par requête.
- Pas de SLO dur sur l'embedding de query isolé phase 1 (dépend du hardware) ; observable via `embedding_ms` du response body.

## Security / trust boundary

- Conforme threat model W-I-1 / `.claude/rules/security.md` A09 : `query` brut et `text` retourné jamais loggés ; seule la longueur `query_len` est journalisée.
- Conforme A05 : message d'erreur DB jamais propagé au client (enveloppe `{"error":"database_unavailable"}` opaque).
- Conforme `.claude/rules/security.md` A04 : limites dures `query ≤ 4 KiB`, `top_k ≤ 20`, timeout SQL 5 s.
- ACL appliquée côté SQL (`WHERE` clause) — un bug applicatif post-recherche ne peut exposer du contenu `members` / `author_only` à un caller `anonymous`.
- `user_tier` est un input de confiance fourni par la gateway interne ; la frontière W-S-1 (auth interne gateway↔workers) est traitée hors de ce ticket. En MVP la gateway envoie toujours `anonymous` (vision §73).
- Endpoint non-public : pas exposé directement par la gateway. Configuration ingress = ticket INFRA-* dédié.

## Observability

- Log JSON par requête : `{"event":"retrieve","query_len":N,"top_k":K,"user_tier":T,"results":R,"embedding_ms":E,"search_ms":S,"status":"..."}`.
- `embedding_ms` et `search_ms` également exposés dans le body de réponse pour debugging direct.
- Pas de métrique OpenTelemetry exportée (ticket OBS-* dédié).
- Pas de trace Langfuse (RET ≠ chemin LLM ; instrumentation traces remontera dans GEN-*).

## Decisions résolues (humain, 2026-05-05)

- A2 — `user_tier` sémantique : **input = scope ACL** mappé directement sur `documents.access_tier`. Valeurs `{anonymous, members, author_only}`. Pas de mapping interne `users.tier → access_tier` dans ce ticket. Note d'écart vs FOUND-003 (où `users.tier ∈ {anonymous, member, author}`) : la gateway/orchestrateur GEN-* est responsable de traduire `users.tier` (identité) → `acl_scope` (portée) avant d'appeler workers. Ce mapping vit dans GEN-002 (cf AC à ajouter).
- B5 — Embedder partagé : module `workers/src/archiviste_workers/embedder.py` (option b). Si RET-001 merge avant ING-001, ce ticket l'introduit ; sinon il l'importe.
- B1 — Client DB workers : **`asyncpg`** (cohérent ING-001 / ING-003), avec codec pgvector (`asyncpg-pgvector` ou enregistrement manuel). ADR `docs/adr/0005-workers-db-asyncpg.md` à introduire dans la PR qui merge en premier.
- B3 — Logger : **`structlog`** (cohérent ING-001 / ING-003).
- OQ-4 — Timeout SQL : **5 s**. Cohérent SLO p95 < 500 ms (10x headroom). Override default A04 (30 s) justifié par profil RAG read-only court.
- OQ-5 — Invariants properties : à enregistrer dans `specs/properties.md` comme `INV-9` (`score ∈ [0,1]`) et `INV-10` (`len(chunks) ≤ top_k`). Modification source de vérité = humain (à faire avant merge).
- OQ-6 — UUID sortie : **canonique RFC 4122** (8-4-4-4-12 hex), cohérent `gen_random_uuid()`.
- OQ-7 — Tie-break score : **SQL-side** `ORDER BY score DESC, chunk_id ASC` (déterminisme garanti DB, pas application).
- OQ-8 — OpenAPI init : ce ticket **initialise** `specs/openapi/gateway-to-workers.yml` si premier endpoint workers (squelette OpenAPI 3.1 + path `/v1/retrieve`). Pas de FOUND-* préalable.
