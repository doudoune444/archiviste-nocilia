## Test strategy
- **Unit** : `test_auth.py` (AC-7 matrix — absence, SA_KEY only, GAC only, both → warning + précédence). `test_main.py` (argparse, exit code AC-19, dry-run exit 0 AC-12).
- **Intégration httpserver** : `test_drive_client.py` (mock Drive endpoints `/files`, `/files/{id}/export`, `/files/{id}?alt=media`, vérifie timeout passé au client, pagination `nextPageToken`).
- **Intégration sync end-to-end** : `test_sync_integration.py` avec `tmp_path` + `DriveClient` stub injectable. Couvre :
  - run 1 (create gdoc + PNG) → fichiers écrits, frontmatter clés ordonnées strictement AC-4, state persisté.
  - run 2 inchangé (`content_signature` identique) → 0 réécriture, `mtime` inchangé, status `unchanged` (AC-11).
  - rename Drive (titre change) → `rename_local_file` appelé, log `image_renamed` ou `status=renamed` (AC-6).
  - suppression Drive → archive idempotente (`archived: true` + `archived_at` figés, second run = `unchanged` sans toucher mtime) (AC-8).
  - collision slug → suffixe `-<id[:8]>` déterministe via tri sur `drive_file_id` (AC-9).
  - body > 1 MiB post-normalize → skip + log error, autres continuent (AC-14).
  - mock 429 → log error, continued, summary `errors >= 1` (AC-15).
  - mock 503 → exit 1, autres OK (AC-16).
  - dry-run sur 4 fichiers (create + update + unchanged + error) → 0 write, statuts `would_*`, exit 0 (AC-12).
  - drift local : édition humaine du `.md` puis Drive change → log `local_drift_overwritten`, overwrite (AC-20).
- **Pas de property test** (pas d'invariant `specs/properties.md` directement applicable au runtime).
- **Pas de contract test** (pas d'OpenAPI touché).
- **Pas d'eval Ragas** (pas de chemin RAG).
- **Firewall AC-14 (ING-010)** : `conftest.py` étendu — autorise imports Drive API uniquement dans `auth.py` + `drive_client.py` (allow-list par filename), continue à bloquer ailleurs.

## Risks / open questions
- **Timing log `creds_conflict`** : émis pré-construction des creds (deux env vars set → log warning unique → continue avec `GDRIVE_SA_KEY_JSON`). Confirmé scope AC-7.
- **Dry-run et `body_hash` drift** : la comparaison locale vs état (AC-20) doit être effectuée en dry-run pour produire un log cohérent ; seul le write est skippé. À acter en revue.
- **Tri déterministe pour AC-9** : la liste Drive n'est pas garantie ordonnée ⇒ on trie en mémoire par `drive_file_id` avant dispatch pour rendre la collision résolution reproductible.
- **`content_signature` vs `last_exported_at`** : seul `drive_md5_checksum` est renommé ; `last_exported_at` (state) et `exported_at` (frontmatter) restent distincts par design.
- **LOC budget** : surface ~340 LOC src + ~150 LOC tests selon spec. Limite vertical-slice 300 LOC src. Si dépassement à l'impl, alerter humain — pas de split prévu ici (sync orchestrator indivisible).
- **`structlog` vs `logging` stdlib JSON** : si `structlog` ajoute trop de surface, fallback `logging` + `json.dumps` manuel suffit pour AC-17. Décision à l'impl.
