# Plan — ING-010 Sync Google Drive utilities (slugify, state, frontmatter merge, paths, normalize, rename)

## Goal
Livrer le package Python `scripts/gdrive_export/` (utilities pures, zéro dépendance Drive API) avec lockfile isolé, hook pre-commit scopé, ADR-0006, prêt à être consommé par ING-013. Aucun runtime, aucun appel réseau, aucun `subprocess` en code de prod.

## Acceptance criteria recap
- AC-1 : package `scripts/gdrive_export/` avec `pyproject.toml` + `uv.lock` isolés ; `cd scripts && uv sync && uv run pytest` passe.
- AC-2 : `slugify(text, fallback_id) -> str` (NFKD → drop combining → lower → non-alnum→`-` → collapse → strip → cap 80 ; fallback `file-<id[:8]>`).
- AC-3 : `slugify` déterministe + idempotent + sortie matche `^[a-z0-9-]+$|^file-[0-9a-f]{8}$`.
- AC-4 : `normalize_body(text) -> str` (NFKC + strip C0 sauf `\n\t`).
- AC-5 : `state.py` — `StateEntry`, `load_state`, `save_state` (JSON canonique trié indent 2), `compute_body_hash` (sha256 hex).
- AC-6 : `merge_frontmatter(existing_yaml, script_managed, defaults_user)` — script-managed écrasés ; user-managed préservés ou défauts ; clés custom intactes ; raise `FrontmatterMergeError` sur YAML invalide.
- AC-7 : `resolve_local_path(...)` — collision déterministe via suffixe `-<id[:8]>`, asserte `is_relative_to(root)`.
- AC-8 : `rename_local_file(old, new)` — `Path.rename()` pur, parents créés, `FileNotFoundError`/`FileExistsError` propagés. Aucun `subprocess`.
- AC-9 : test intégration repo git tmp → `git status --porcelain` post-rename affiche `R` (similarity ≥ 50 %).
- AC-10 : `cd scripts && uv run ruff check . && uv run mypy .` exit 0.
- AC-11 : `.pre-commit-config.yaml` hook local scopé `^scripts/.*\.py$`.
- AC-12 : `.gitignore` étendu (`.gcp/`, `scripts/.gdrive_state.json`, `scripts/.gdrive_state.json.bak`).
- AC-13 : ADR-0006 status `accepted`.
- AC-14 : grep CI confirme aucun import Drive API dans `scripts/gdrive_export/`.
- AC-15 : mypy strict + docstrings + fn ≤ 40 lignes.

## Security posture
- **Lockfile isolé** : `scripts/pyproject.toml` + `scripts/uv.lock` distincts du venv `workers/`. Aucune dep ajoutée à `workers/` ou `gateway/`. Cf ADR-0006 : blast radius zéro runtime sur l'app de production.
- **Aucune Drive API ce ticket** : déféré ING-013. Vérifié par grep CI (AC-14). Permet de shipper la lib sans approbation cloud externe.
- **Aucun `subprocess` en code prod** (AC-8 override OQ-N) : zero argv surface élimine toute injection shell par construction. `subprocess` apparaît uniquement dans `test_rename.py` comme oracle lecture seule (`git status --porcelain`).
- **Path traversal** : slugification systématique de chaque composant + assertion `is_relative_to(root)` dans `paths.py` (AC-7). Tentative `..` slugifie vers fallback safe.
- **Secret hygiene** : aucun secret manipulé dans ce ticket, `.env.example` non modifié (déféré ING-013). `.gitignore` ajoute `.gcp/` (futur dossier creds local jamais commité).
- **Pre-commit local hook** : ruff + mypy strict scopés `^scripts/` empêchent toute régression typing/style avant push.

## Risks / open questions
- **`pyyaml` strict mode** : `yaml.safe_load` accepte les YAML non-canonical (booléens implicites `yes`/`no`, dates auto). Pour le frontmatter, on veut stricte interprétation des strings : si un humain écrit `access_tier: yes` en local, on doit préserver `yes` literal, pas `True`. Décision plan : `yaml.SafeLoader` standard suffit ; les tests AC-6 doivent couvrir un cas `tags: [yes, no]` pour valider que round-trip ne casse pas. Si problème → `pyyaml` resolver custom (mineur) ou `ruamel.yaml` (lourd, ADR séparé).
- **`pytest-httpserver` non requis ce ticket** : pas d'appel réseau ⇒ pas besoin. Confirmé : reportée à ING-013 deps.
- **Path Windows** : `Path.rename()` est cross-platform. Le test `git status --porcelain` produit des séparateurs `/` même sur Windows (git normalise) ⇒ oracle stable.
- **`scripts/.gdrive_state.json` ignoré par `.gitignore` ce ticket vs versionné en ING-013** : volontaire — ce ticket ne livre pas de runtime qui produit le fichier ; l'ignore évite tout fichier orphelin créé pendant tests. ING-013 amendera `.gitignore` pour retirer la ligne au moment où le state file devient versionné par run.
- **Hook pre-commit local mypy lent** : sur gros change, mypy peut prendre 5-10s. Acceptable phase 1, à ré-évaluer si friction dev.

## Pre-flight (architect, 2026-05-09)
- (a) **Fichiers/dirs lus** : `specs/acceptance/ING-010.md` (avant et après amendement), `specs/plans/ING-010.md` (ancien), `docs/adr/0000-template.md`, `docs/adr/` listing, `.gitignore`, `.pre-commit-config.yaml` (présence confirmée), `.claude/rules/clean-code.md`, `.claude/rules/no-workaround.md`, `.claude/rules/secret-hygiene.md`, `.claude/rules/security.md`, `.claude/rules/vertical-slice.md`, `CLAUDE.md` racine.
- (b) **3 hypothèses clés** :
  1. `pyyaml` `SafeLoader` est suffisant pour préserver les clés custom du frontmatter sans coercion implicite gênante (validé par tests matriciels AC-6).
  2. `Path.rename()` cross-platform Windows/Linux fonctionne sur fichier suivi git ⇒ git détecte le rename via similarity ≥ 50 % au prochain `git status` (pas de mécanisme spécifique requis côté Python).
  3. La séparation lockfile `scripts/uv.lock` ↔ `workers/uv.lock` est acceptable pour la maintenance long-terme (un `uv sync` distinct par directory) et préférable à une dep partagée pour limiter blast radius (cf ADR-0006).
- (c) **Zones d'incertitude** :
  1. Convention exacte de `compute_body_hash` : sha256 sur bytes UTF-8 du corps Markdown brut (post-normalize NFKC) ou pre-normalize ? Plan choisit **post-normalize** pour cohérence avec ce qui sera écrit sur disque ; ING-013 doit rappeler cette convention. À valider humain.
  2. `merge_frontmatter` doit-il préserver l'ordre des clés du YAML existant (ruamel.yaml requis) ou peut-il ré-ordonner alphabétiquement (pyyaml dump suffit) ? Plan choisit **ré-ordonnancement alphabétique** (déterministe, diff stable) ; à confirmer humain (impact diff lisibilité).
  3. Le hook pre-commit local doit-il `cd scripts && uv run mypy .` (lent, isolation parfaite) ou directement `mypy --config-file scripts/pyproject.toml scripts/` (rapide, mais nécessite mypy global) ? Plan choisit **`uv run`** pour cohérence avec workflow dev. À valider humain.
