## OQ-A resolution — `<ingest_root>` + download wiring
- `<ingest_root>` = `/tmp/lore` (writable tmpfs in the Job container; ephemeral, destroyed at execution end — security §"download éphémère"). Download target = `/tmp/lore/lore/`. Root path read from env `INGEST_ROOT` (default `/tmp/lore`), bucket from `LORE_CORPUS_BUCKET` — no magic constants in code.
- Wiring = **a small dedicated module** `archiviste_workers.ingest.download_corpus` run as its own process, then the CLI, chained by a shell `command` in the Job. Rejected alternatives: inline `python -c` (untestable, lint-hostile, escaping nightmare), `gcloud storage rsync` in Job (no gcloud in workers image; spec forbids adding it). The Job `command` becomes a `bash -c` sequence: `python -m archiviste_workers.ingest.download_corpus && python -m archiviste_workers.ingest --root /tmp/lore --path /tmp/lore/lore/` using the `/app/.venv/bin/python` interpreter. The downloader is a pure, unit-testable function (list blobs → write each to `<root>/lore/<name>`, mkdir parents) — SRP, mockable `Client`.
