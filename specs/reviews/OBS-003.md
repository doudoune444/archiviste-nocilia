## Control-flow correctness
- Insertion point: persist runs after `write_run` and before `apply_gate_a` / `_handle_auto_create_baseline` (ragas_runner.py:455-462). On success or skip, `_maybe_persist` returns None → EVAL-001 verdict flow (gate_a, auto-baseline, summary) reached **unchanged**. On failure, returns 4 early — intentional dedicated code (D-7), does not corrupt artifact (already written) and does not run gates. No path inserts a NULL row: null/absent metrics short-circuit in `_build_eval_run_row` (returns None) → exit 4 before any connect. Auto-baseline branch only reached when persist returns None, so no interaction defect. Exit 4 is never swallowed.

## Security
- No secrets in code or tests (test DSNs are fixtures with fake passwords; secret-hygiene §Tests compliant).
- A03 Injection: INSERT is fully parameterized; `_INSERT_SQL` is a static literal, values bound positionally. No `format!`/f-string/`%`-interp on the query. ✓
- A09 Logging/secrets: `DATABASE_URL` read no-default (`os.environ["DATABASE_URL"]`, persist.py:77 — KeyError if absent, no fallback). Never serialized to run-file/stdout/stderr/log, including failure path. `persist_failed` log emits no exc/exc_info. PersistError caught locally, never propagates uncaught. ✓ Note: `DATABASE_URL` is handled as a plain `str` rather than `pydantic.SecretStr`; given it is never logged/serialized and lives only as a local in one function, this is acceptable, but `SecretStr` would be defense-in-depth (LOW, non-blocking).
- D-8 driver reuse: `psycopg2-binary` pre-declared; no new dependency, no ADR needed. ✓
