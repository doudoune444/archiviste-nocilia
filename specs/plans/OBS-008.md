# Plan — OBS-008 Eval runner container image (CI build + Artifact Registry push)

## Pre-flight

(a) Read: `specs/acceptance/OBS-008.md`; `infra/docker/workers.Dockerfile`, `gateway.Dockerfile`;
`eval/pyproject.toml`; `eval/ragas_runner.py` (argparse, `main`, `__main__` guard, imports);
`eval/__init__.py` (empty); `.github/workflows/deploy.yml`; Glob of `eval/**`, `infra/docker/**`,
`.github/workflows/*.yml`; confirmed no existing `.dockerignore`.

(b) Key assumptions:
1. Build context `eval/` + `COPY . /app/eval/` + `WORKDIR /app` makes `import eval.*` and
   `python -m eval.ragas_runner` resolve with no editable install (plain `sys.path` = `/app`).
2. `--mode` is `required=True` but argparse prints `--help` and exits 0 *before* validating it, with
   no network/git side-effects at import time → AC-1/AC-5 (`--network none --help`) hold as-is.
3. `ragas`/`datasets`/`langchain-openai` move from `dev` to a new `live` extra so the image installs
   live deps without pytest/ruff/mypy; this regenerates `eval/uv.lock` deterministically.

(c) Uncertainty zones:
- `uv sync --extra live` inside a context-root-only build (no repo `.git`) — `_get_git_sha` returns
  `"unknown"` gracefully (try/except), so no git needed in image. Verify install command exact form.
- Whether `uv sync` builds the `eval` wheel (hatch `packages=["eval"]`) or just installs deps;
  we rely on raw `COPY` for source, deps-only install — confirm `--no-install-project` is acceptable.
- Image must run `python` directly (not `uv run`, which may re-resolve) for offline guarantee.

## Goal
Ship a reproducible Docker image of the Ragas eval runner (code + `--frozen` locked deps +
`baseline.json`), built and pushed to Artifact Registry `archiviste/eval:{sha}` + `:latest` on every
merge to `main`, for OBS-009 to deploy as a Cloud Run Job.

## Acceptance criteria recap
- AC-1 : `infra/docker/eval.Dockerfile`, context `eval/`, ENTRYPOINT invokes `python -m eval.ragas_runner`; `docker run <image> --help` exits 0.
- AC-2 : bakes (a) runner code, (b) deps from `eval/uv.lock` `--frozen`/`--locked`, (c) `eval/baseline.json`; does NOT bake `specs/golden_qa.jsonl`.
- AC-3 : `--output`/`--persist` stay runtime args; no output/persist default baked.
- AC-4 : full live deps (Ragas + OpenAI judge + Mistral generate) but NO local `bge-m3`/`sentence-transformers`/`torch`.
- AC-5 : `docker run --network none <image> --help` exits 0; no runtime `pip install`/`uv sync`.
- AC-6 : CI builds on push `main`, pushes `${REGION}-docker.pkg.dev/${PROJECT}/archiviste/eval` tags `:${sha}` + `:latest`, reusing repo `archiviste`.
- AC-7 : step exits 0 iff build AND push of both tags succeed; any error fails the job.
- AC-8 : no new Terraform resource; repo `archiviste` reused, no new IAM.
- AC-9 : `specs/openapi/gateway-to-workers.yml` unchanged.
- AC-10 : `migrations/*.sql` unchanged.

## Open-question decisions
- **OQ-1 → extend `deploy.yml`.** It already has WIF auth, `gcloud auth configure-docker`, buildx, and
  the build-push pattern with `:sha`+`:latest` (workers). Adding a 3rd `build-push-action` step reuses
  all setup, single image-publish path, mirrors `workers:latest`. No dedicated `eval-image.yml`.
  AC-6 names `<workflow>` without pinning it.
- **OQ-2 → `COPY . /app/eval/` + `WORKDIR /app`, no editable install.** Build context is `eval/`, whose
  contents are the `eval` package body. Copying the context into a dir literally named `eval/` under
  `/app` (on `sys.path`) makes `import eval.ragas_runner` resolve directly. Deps installed to a venv;
  source served by raw COPY (not wheel) → no rebuild on code change, simplest, no `PYTHONPATH` edit.
- **OQ-3 → add `live` extra to `eval/pyproject.toml`; move `ragas`/`datasets`/`langchain-openai` `dev`→`live`.**
  `[project.optional-dependencies] live = ["ragas>=0.2","datasets>=3.2","langchain-openai>=0.2"]`;
  keep them ALSO referenced by `dev` (so PR/CI lint+test path unchanged) via `dev = [...tools..., {include live}]`
  pattern OR list them in both. Dockerfile installs `--extra live --no-dev`. Regenerates `eval/uv.lock`.
  **HUMAN-ONLY FILE: `eval/uv.lock` regen + `eval/pyproject.toml` edit require explicit human OK** (flagged below).

If human rejects moving deps, fallback = Dockerfile installs `--all-extras` (heavier: bakes pytest/ruff/mypy) — violates the "minimal image" intent of AC-4's spirit but still meets the literal AC.
