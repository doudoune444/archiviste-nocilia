# Plan — OBS-003 Persistence Postgres des runs Ragas live (table eval_runs + flag --persist)

## Goal
En `--mode live --persist`, le runner Ragas insère une ligne agrégée par run dans la nouvelle table `eval_runs` (write direct paramétré, artefact-puis-persist), débloquant la lecture OBS-004.

## Acceptance criteria recap
- AC-1 : migration crée `eval_runs` avec exactement les colonnes contractuelles (id UUID PK, git_sha, runner_mode CHECK='live', golden_set_version, 4 métriques NUMERIC(5,4) NOT NULL, entries_total/ok/errors INT, started_at/finished_at/created_at TIMESTAMPTZ).
- AC-2 : index partiel `eval_runs_live_latest_idx ON eval_runs (finished_at DESC) WHERE runner_mode='live'`.
- AC-3 : flag `--persist` (store_true) ; sans lui, zéro accès Postgres, comportement byte-identique EVAL-001.
- AC-4 : `--persist` + `--mode live` réussi → exactement 1 ligne, colonnes == champs homologues du run-file.
- AC-5 : `--persist` + `--mode offline` → 0 insert, log `event=persist_skipped reason=offline_mode`, exit inchangé.
- AC-6 : `golden_set_version` = SHA-256 du contenu d'octets de `specs/golden_qa.jsonl` ; déterministe.
- AC-7 : append-only, pas d'UPDATE/UPSERT/dedup ; 2 persists → +2 lignes, `id` distincts.
- AC-8 : INSERT paramétré (placeholders liés), aucune interpolation de valeur dans le SQL.
- AC-9 : aucun payload LLM (answer/retrieved_contexts/citations/question) en DB.
- AC-10 : `DATABASE_URL` jamais sérialisé (run-file/stdout/stderr/logs), y compris chemin d'échec.
- AC-11 : échec INSERT → artefact intact, log `warn event=persist_failed` sans secret ni stack DB, exit `4`.
- AC-12 : écriture run-file précède la tentative de persist (artefact-puis-persist).
- AC-13 : `specs/openapi/gateway-to-workers.yml` non modifié.

## PROPOSED SQL — `migrations/0005_eval_runs.sql`
> **PROPOSED — human-only. L'agent NE rédige PAS ce fichier.** L'humain l'écrit/valide avant impl (CLAUDE.md §Sources de vérité). Filename matche `run.sh` regex `^[0-9]{4}_[a-z0-9_]+(\.up)?\.sql$`. Première ligne = directive `-- description:`.
```sql
-- description: OBS-003 — eval_runs (1 ligne agrégée/run live) + index partiel live-latest
CREATE TABLE eval_runs (
    id                 UUID PRIMARY KEY,
    git_sha            TEXT NOT NULL,
    runner_mode        TEXT NOT NULL CHECK (runner_mode = 'live'),
    golden_set_version TEXT NOT NULL,
    faithfulness       NUMERIC(5,4) NOT NULL,
    answer_relevancy   NUMERIC(5,4) NOT NULL,
    context_precision  NUMERIC(5,4) NOT NULL,
    context_recall     NUMERIC(5,4) NOT NULL,
    entries_total      INT NOT NULL,
    entries_ok         INT NOT NULL,
    entries_errors     INT NOT NULL,
    started_at         TIMESTAMPTZ NOT NULL,
    finished_at        TIMESTAMPTZ NOT NULL,
    created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX eval_runs_live_latest_idx
    ON eval_runs (finished_at DESC)
    WHERE runner_mode = 'live';
```

## Insertion point & control flow (`main()`)
Après `write_run(output_path, run)` (artefact écrit+fermé, AC-12), avant `gate_a = apply_gate_a(...)` :
```
persist_exit = _maybe_persist(args, run, runner_mode)
if persist_exit is not None:
    return persist_exit      # 4 sur échec live (AC-11)
# fallthrough → verdict gates inchangé
```
`_maybe_persist` :
- `not args.persist` → `return None` (no-persist fallthrough, zéro import psycopg2 touché, AC-3).
- `runner_mode == "offline"` → `log.info("persist_skipped", reason="offline_mode")` ; `return None` (AC-5, verdict gate inchangé).
- live + persist → `golden_set_version = derive_golden_set_version(args.golden_set)` ; build `EvalRunRow` depuis `run` (run.runner_mode → colonne `runner_mode`, c3) ; `try: persist_eval_run(row); log.info("persist_ok", id=..., golden_set_version=..., finished_at=...)` → `return None` (succès, verdict gate inchangé) ; `except PersistError: log.warning("persist_failed")` (sans détail/secret) → `return 4` (AC-11).

## persist module — connexion, redaction, mypy
- `DATABASE_URL` : `os.environ["DATABASE_URL"]` (KeyError si absent — **no default**, secret-hygiene §Production). Lu dans `persist_eval_run`, jamais stocké en attribut sérialisé.
- INSERT paramétré : un seul littéral SQL constant avec `%s` placeholders ; valeurs passées en tuple à `cursor.execute(SQL, params)`. Aucune f-string/`.format(`/`%`-formatting sur le SQL (AC-8). `id` = `str(uuid.uuid4())` côté Python.
- Redaction : aucune exception persist ne propage `DATABASE_URL`. Le log `persist_failed` n'émet aucun champ connexion. Réutilise `SECRET_ENV_VARS`/`_redact_raw` de `eval/run_writer.py` uniquement si une chaîne d'erreur doit être loggée (par défaut : message générique, pas de `str(exc)` brut). `PersistError(Exception)` typée locale.
- mypy `--strict` : psycopg2 untyped → `typing.cast()` / annotations locales explicites sur `connect`/`cursor` returns (c2). **Jamais** `# type: ignore` (no-workaround).
- `EvalRunRow` (dataclass) : champs = colonnes non-`created_at`/non-`id`, tous métriques+métadonnées, **aucun** `answer`/`question`/`contexts`/`citations` (AC-9, illegal-state-by-construction).
