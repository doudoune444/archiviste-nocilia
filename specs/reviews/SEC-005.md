## Security

Mapped against `.claude/rules/security.md`. No HIT.

- Secrets in code: none. Mock tokens (`test-sql-token`, `cached-token`, `fresh-token`, `stale-token`, `cache-hit-token`, `sql-token`, `gcs-token`) are fixture literals only.
- Sensitive types: bearer wrapped in `SecretString` end-to-end; `expose_secret()` calls only at `PgConnectOptions::password` (`lib.rs:84, 140`) — never logged.
- SQL injection: no new SQL.
- SSRF: metadata URL hardcoded `http://metadata.google.internal`. Injectable base URL only via `with_base_url(...)` used by tests. No user input.
- Trust boundary: gateway computes the SQL token itself, no client header trust.
- A09 logging: refresher emits `error = %e` — `reqwest::Error` Display safe (no headers/body); `TokenError` enum variants are static strings. No bearer leakage.
- CORS / CSP / HSTS / JWT / rate limit / embedding poisoning / prompt injection / LLM XSS / unsafe deserialization: not touched.
