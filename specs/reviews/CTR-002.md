## Security

- A01 identity spoofing / IDOR: PASS. `x-user-tier`/`x-user-id` forwarded to workers are read from `AnonIdentity` extension (report_contradiction.rs:222-223), computed by `resolve_identity` middleware ŌĆö never from request body or inbound client headers. Outbound reqwest sets them fresh. Citations re-resolved ACL-bounded server-side in workers (CTR-001), so a client cannot forge source bodies.
- request_id not client-trusted: PASS. Always `req_id.0` from `attach_request_id` middleware (handler:69); client body `request_id` never read. Sent to workers as `x-request-id` header + JSON `request_id` from the gateway value (handler:213, 221).
- A03 / claim handling: PASS. `claim` sent as JSON only (`json!` macro, handler:210), never interpolated into URL/markup/SQL. Logged as `claim_len` only (handler:81); `log_request` never receives the string (A09 satisfied).
- Output sanitization (XSS): PASS. Frontend renders all text via `textContent` (button label, textarea, feedback strings). Claim sent as JSON; feedback strings are static literals, never echo upstream content. No `innerHTML` in the new code.
- Error envelope: PASS. `build_error` returns `{error, request_id}` only. Upstream 4xx/5xx collapse to `upstream_error` 502 (`ac4_workers_400/500` confirm no upstream-body passthrough). `application/json; charset=utf-8` set. Frontend never surfaces envelope internals (generic retry on non-200).
- A04 DoS: PASS (enforcement + tested). Route body cap 1 MiB (`RequestBodyLimitLayer`, lib.rs:506) + upstream cap 256 KiB (`read_capped_body`, streaming reject). Validation bounds (claim 1..4096, citations 1..50, chunk_ords Ōēż200, non-negative) all enforced in `parse_and_validate` BEFORE the ID-token fetch + workers fan-out. Ordering correct; cap now has a test.
- No `unwrap`/`expect`/`panic` on user input: PASS. Only `unwrap_or_else` is `build_passthrough` (workers_proxy.rs:77) on the infallible `Response::builder` with a static fallback. `elapsed_ms` uses `unwrap_or(i64::MAX)`. ID-token via `secrecy::ExposeSecret`. (Test-file `unwrap`s are gated `#![allow(clippy::unwrap_used)]` per project test convention.)
- No secrets in code: PASS. None detected.
