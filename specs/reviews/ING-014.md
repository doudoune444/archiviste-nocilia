## Security

- No secrets in diff (verified by grep `AKIA|xox[baprs]|gh[pousr]_|ya29|eyJ`).
- Service account scope `documents.readonly` correctly added to `auth.py:_SCOPES` (line 25).
- `download_image` uses authorized httplib2 transport — same SA, no separate cred path.
- Token never logged: `download_image` does not log header or response body. OK.
- SSRF: `download_image` accepts any URI without scheme/host validation. Documented exception in spec L120 (`*.googleusercontent.com` Google-controlled, mono-author trust boundary). Acceptable per spec, but a single line in spec leaves a future risk if `contentUri` ever points elsewhere — log the resolved host as info would aid forensics. Not a blocker.
- Path traversal: AC-18 enforced in `compute_image_path` (md5 regex + `is_relative_to`). `cleanup_orphans` lacks the same guard (LOW above).
- DecompressionBomb: `PIL.Image.MAX_IMAGE_PIXELS = 2048*2048*4` set at module load. No test coverage (LOW above).
- ADR-0007 Pillow FFI: correctly created, references `.claude/rules/security.md` A06, lists alternatives.
- MD5 use: `hashlib.md5(...)` used for filename derivation only (not crypto). `# noqa: S324` correctly applied at sync.py:885 + images.py:133.
