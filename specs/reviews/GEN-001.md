## Summary

The rebase conflict in `main.py` is correctly resolved (both routers mounted, `create_pool` for pgvector preserved, `http_client` lifecycle clean, fail-fast LLM_* at boot). The previous out-of-scope `specs/acceptance/RET-001.md` is no longer in the diff thanks to the rebase.

However, the new `fix(infra)` commit introduces a HIGH-severity CI breakage: `docker-compose.yml` now requires `LLM_PROVIDER`, `LLM_MODEL`, `LLM_API_KEY` to be set non-empty for ANY `docker compose` invocation that interpolates the `workers` service (`config`, `build`, `up`). The `boot-sla.yml` workflow's two jobs (`measure` and `stack-integration`) write a `.env` that does not include any `LLM_*` variable, so both will fail at the first `docker compose` step. Reproduced locally with `REDIS_PASSWORD=test docker compose config workers` → `LLM_MODEL required`. AC-8 mandates fail-fast at the worker process level (already covered by `LlmClient.from_env()`); compose-level enforcement is an additional, untested constraint that breaks existing infrastructure tests.

Additionally, the lifespan addition silently disables the RET-001 pgvector regression test (`test_main_lifespan.py`) because `LlmConfigError` is swallowed by an over-broad `except RuntimeError`, leaving the test always-skipped in CI without `LLM_*` env. This is an unintended consequence of the rebase, not gaming, but it must be fixed: either set placeholder env in the test fixture or narrow the skip exception.

Both new findings are objective regressions caused by this branch and must land before merge. Carry-over MED/LOW findings remain acceptable per OQ-8 / prior review acceptance.

**Verdict: REQUEST_CHANGES.** Two items to address:
1. Either soften `${LLM_*:?...}` to `${LLM_*:-default}` in `docker-compose.yml` (preferred — keeps process-level AC-8 fail-fast as the single source of truth) or update `boot-sla.yml` (and any other compose-using workflow) to set placeholder `LLM_*` env vars.
2. Fix `test_main_lifespan.py` so the RET-001 pgvector regression is actually exercised after this branch lands (set `LLM_*` via `monkeypatch.setenv` and narrow the skip exception).
