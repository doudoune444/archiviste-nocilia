### (b) Three key hypotheses

1. **PR-e GCS placeholder is inline literal, NOT a Secret Manager resource.** Confirmed by reading `.worktrees/INFRA-002-pr-e/specs/plans/INFRA-002-pr-e.md` §"HCL snippets" lines 157-165: PR-e adds an `env { name = "GCS_SIGNING_PRIVATE_KEY_PEM" value = "placeholder-removed-by-SEC-004" }` block directly in `cloud_run.tf`, with **no** `google_secret_manager_secret.gcs_signing_private_key` resource. Spec AC-8 wording ("Le secret Manager `gcs_signing_private_key` (créé par INFRA-002 PR-e comme placeholder) est supprimé via un bloc `removed`") is **factually wrong about provisioning**. The Terraform `removed` block has nothing to remove. Resolution: AC-8 collapses to deleting the inline env block in `cloud_run.tf` only — flagged in Risks for human confirmation before impl.
2. **Shared `reqwest::Client` exists in `AppState.http`** (state.rs L25) with `connect_timeout = Config.connect_timeout_ms` and `timeout = Config.request_timeout_ms` (default 5 s / 35 s). AC-6 requires **dedicated** timeouts (2 s / 5 s) for the signing path → cannot reuse `state.http`. Plan: `TokenProvider` owns its own `reqwest::Client` built once at `AppState::new` with `connect_timeout(2s)` + `timeout(5s)` hardcoded constants in `sign.rs`.
3. **`tokio::time::Instant` + injectable clock-base is unnecessary** — `CachedToken` stores `expires_at: chrono::DateTime<Utc>` computed at fetch time as `Utc::now() + Duration::seconds(expires_in)`. Refresh-ahead test (AC-3 oracle, AC-9 sub-case g/h) does NOT need a fake clock: it controls `expires_in` from the mock metadata response (e.g. `expires_in=1` to force immediate refresh on second call, or `expires_in=3600` to assert cache-hit `expect(1)`). This is cleaner than introducing a `Clock` trait per `clean-code.md` §"No premature abstraction".

### (c) Zones of uncertainty

- **AC-8 discrepancy** (see hypothesis 1): is the spec's "Terraform `removed` block" mandate vestigial wording from Round 2 (when secret-resource provisioning was assumed) or a genuine requirement? If genuine, human must clarify which resource to remove. Default: plan treats AC-8 as "delete inline env block; no `removed` block needed because no resource exists" and flags loudly.
- **Test fake-clock for refresh-ahead semantics** (AC-3): `expires_at - 60s` window means a token with `expires_in=61` is "fresh" for ~1 s. Testing strategy below uses **mock-controlled `expires_in`** (1 vs 3600) rather than fake clocks — simpler, no new trait, matches PR-e §"Verified safe: …no validation" minimalism principle.
- **Whether `dev-deps rsa` / `pkcs8` / `rand_core` removal breaks `common/jwt_helpers.rs`**: `jwt_helpers.rs` uses `pkcs8::{EncodePrivateKey, EncodePublicKey}` + `rand_core::OsRng` for Ed25519, NOT for RSA. So `pkcs8` + `rand_core` MUST stay (Ed25519 path), only `rsa` may be removed. `cargo tree` post-impl confirms.
- **`reqwest` `mockito` URL injection**: `TokenProvider` and `sign_get` must accept a base URL override (or accept a pre-built reqwest client pointing at mockito). Plan: `TokenProvider::new(metadata_base_url, iam_base_url)` takes two `String` args, defaults provided by `AppState::new` to the real Google endpoints; tests pass mockito `server.url()` for both.

## Goal

Replace the offline RSA-PKCS1-SHA256 signing in `gateway/src/gcs/sign.rs` with an HTTP call to `iamcredentials.googleapis.com/v1/.../{sa_email}:signBlob` authenticated by a Cloud Run metadata-server bearer token. Drops the SA private key from gateway config (AC-1, AC-11), introduces a dedicated `TokenProvider` struct with refresh-ahead caching and retry-on-401 (AC-3), maps 6 failure modes to a discriminated `reason_code` log field (AC-5), and adds the `roles/iam.serviceAccountTokenCreator` self-binding in Terraform (AC-7). REST contract `GET /v1/conversations/{id}/signed-url` unchanged (Non-goals).

## Risks / open questions

- **AC-8 discrepancy (BLOCKER until human confirms)**: spec says "secret Manager `gcs_signing_private_key` (créé par INFRA-002 PR-e comme placeholder) est supprimé via un bloc `removed` Terraform". Confirmed via PR-e plan: PR-e creates **no** `google_secret_manager_secret.gcs_signing_private_key` — the GCS placeholder is an **inline literal string** in `cloud_run.tf`. Therefore the Terraform `removed` block has no resource to target. **Two options**: (1) human confirms AC-8 collapses to "delete inline env block only, no `removed` block needed" (preferred — matches reality), or (2) human amends spec / re-opens an OQ. Plan provisions for option 1 by default ; step 15 is gated on this clarification.
- **Shared `reqwest::Client` decision**: `AppState.http` exists with general 5 s/35 s timeouts; AC-6 requires dedicated 2 s/5 s for the signing path. **Resolution**: `TokenProvider` owns a separate `reqwest::Client` built once in `AppState::new` ; `sign.rs` reuses the same client via `token_provider.client_ref()` accessor (no second client). This minimises pool count (one extra) while satisfying AC-6 hardcoded constants. ~3 LOC of `state.rs` plumbing.
- **Fake clock for refresh-ahead**: Plan uses **mock-controlled `expires_in`** values (1 s vs 3600 s) rather than introducing a `Clock` trait. AC-3 oracle accepted because the cache check is `Utc::now() >= expires_at - 60s` — with `expires_in=1`, the token is "stale" within the first call window, forcing refresh on the second call. No need to mock wall-clock. Trade-off: requires `tokio::time::sleep(Duration::from_millis(100))` between the two calls in `sec004_token_refresh_ahead` to let `Utc::now()` advance past `expires_at - 60s` (which is `Utc::now() - 59s` at token-fetch time, so already past — no sleep actually needed). Cleanest approach per `clean-code.md`.
- **Boot-time TokenProvider construction**: `AppState::new` becomes fallible if `reqwest::Client::builder()` errors (already fallible — returns `Result`). No new failure mode.
- **`ring` removal**: requires verification post-refactor that no other module uses `ring::signature`. Current code: `sign.rs` is the sole user (`use ring::signature::{RsaKeyPair, RSA_PKCS1_SHA256}`). Removing `ring` reduces compile time + binary size. Confirm via `cargo tree`.
- **`pkcs8` + `rand_core` dev-deps retained** (Ed25519 path in `jwt_helpers.rs`) — only `rsa` dev-dep removed.
