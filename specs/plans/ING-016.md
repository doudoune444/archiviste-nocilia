# Plan — ING-016 Chunker tokenizer swap (BGE-M3 → Mixtral-8x7B-v0.1)

## Goal
Drop `transformers` from the workers runtime by loading a vendored Mixtral-8x7B-v0.1 `tokenizer.json` via the already-present `tokenizers` (Rust) lib to size the chunker splitter, killing the Cloud Run ingest Job OOM driver.

## Acceptance criteria recap
- AC-1 : `workers/src/archiviste_workers/ingest/chunker.py` n'importe plus `transformers` (aucune occurrence de `import transformers` ni `from transformers`).
- AC-2 : Le splitter mesure `chunk_size`/`chunk_overlap` via le tokenizer **Mixtral-8x7B-v0.1** chargé avec la lib `tokenizers` (Rust, déjà présente via `langchain-mistralai`) ; aucune nouvelle dépendance runtime ajoutée à `[project.dependencies]`.
- AC-3 : L'init du chunker n'effectue aucune requête réseau ; tokenizer chargé depuis un `tokenizer.json` vendoré (committé, Apache-2.0, baké au build), pas de fetch HF, pas de HF token. (Vérifié par test : init réseau coupé / HTTP mock asserte zéro requête sortante.)
- AC-4 : Le chunker découpe un document fixture en chunks respectant `chunk_size=512` / `chunk_overlap=64` mesurés en tokens Mixtral, ordre de séparateurs existant. (Correction pure ; aucune comparaison BGE-M3.)
- AC-5 : `[project.dependencies]` ne contient plus `transformers` ; `transformers` absent de toute table de dépendances (runtime / dev / optional).
- AC-6 : Bloc `[project.optional-dependencies] embedder-fallback` supprimé ; `sentence-transformers` absent de toute table.
- AC-7 : L'image workers runtime perd ≥ 50 MB (`docker images archiviste-workers --format '{{.Size}}'` avant/après).
- AC-8 : Gate CI Ragas standing (`eval/ragas_runner.py` sur `specs/golden_qa.jsonl`) reste vert ; aucun nouveau seuil ticket-spécifique.

## Risks / open questions
- HUMAN PREREQUISITE: vendored `tokenizer.json` is gated on HF (`mistralai/Mixtral-8x7B-v0.1`). Agent cannot fetch it (no HF token, no network fetch allowed). If absent at impl time → blocker per `no-workaround.md`, stop. Plan assumes the human commits it first.
- Asset packaging: hatchling wheel by default ships only `.py`. The `.json` must be force-included or it is absent at runtime in the image → AC-3 `RuntimeError`. Verify `uv sync --no-dev` wheel install carries it.
- Identity assert robustness: vocab_size 32000 + EOS id 2 is my proposed pin (Mistral SP-BPE family). If the committed artifact differs (e.g. v0.3 with extended vocab), the identity test must be reconciled to the actual committed file — flag for reviewer.
- `RecursiveCharacterTextSplitter` `length_function` path is transformers-free and confirmed; `from_huggingface_tokenizer` is NOT usable (requires transformers) — do not use it.
- AC-4 overlap semantics: `RecursiveCharacterTextSplitter` overlap is best-effort, not an exact 64-token guarantee on every boundary. Test asserts ≤512 hard and order-preservation; overlap asserted as "non-zero where chunks split", not exact 64.
