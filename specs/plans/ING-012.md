# Plan — ING-012 Workflow `gdrive-sync.yml` + auto-PR

## Goal
Livrer un workflow GitHub Actions (cron hebdo lundi 03:00 UTC + `workflow_dispatch`) qui invoque `scripts/gdrive_export`, capture le summary JSON, et ouvre une PR auto vers `main` (draft si erreurs), avec `actionlint` câblé pre-commit + CI.

## Implementation steps (ordered)
1. **Pre-commit hook** : ajouter `rhysd/actionlint` à `.pre-commit-config.yaml` pinné SHA. Vérifier `pre-commit run actionlint --all-files` localement.
2. **CI step actionlint** : modifier `.github/workflows/ci.yml` job `lint` → ajouter step `name: actionlint` après `check port consistency`, utilisant `rhysd/actionlint@<SHA>` (action officielle, pas `run:`).
3. **Workflow gdrive-sync.yml** :
   a. Skeleton `name`, `on: { workflow_dispatch, schedule: [cron: '0 3 * * 1'] }`, `concurrency`, top-level `permissions: { contents: write, pull-requests: write, actions: read }` (autres implicites `none` via doc YAML comment).
   b. Job `sync` : `runs-on: ubuntu-24.04`, `timeout-minutes: 30`.
   c. Step 0 `check secret`: `if [ -z "${{ secrets.GDRIVE_SA_KEY }}" ]; then echo "secret GDRIVE_SA_KEY missing — see docs/runbook.md" >&2; exit 1; fi` (AC-13, avant checkout).
   d. Step `checkout` : `actions/checkout@<SHA>` `fetch-depth: 0`.
   e. Step `setup uv` : `astral-sh/setup-uv@<SHA>`.
   f. Step `sync deps` : `cd scripts && uv sync`.
   g. Step `run sync` : `cd scripts && uv run python -m gdrive_export --root-folder-id "${{ vars.GDRIVE_ROOT_FOLDER_ID }}" | tee /tmp/sync.log` ; env `GDRIVE_SA_KEY_JSON: ${{ secrets.GDRIVE_SA_KEY }}` au scope step uniquement. **NB** : entrypoint forme module `gdrive_export` (cohérent `__main__.py` + runbook), pas `scripts/gdrive_export.py` comme écrit dans AC-5 → voir Risks.
   h. Step `extract summary` : `jq` sur la ligne `gdrive_sync.summary` de `/tmp/sync.log` → écrit `summary.json` (artifact) + outputs `summary_line`, `errors`, `total`, `created`, `updated`, `renamed`, `archived`, `unchanged` via `>> $GITHUB_OUTPUT`.
   i. Step `detect diff` : `git status --porcelain lore/ scripts/.gdrive_state.json` → output `has_changes`. Si vide : `echo "no changes to sync"` et skip création PR (AC-6).
   j. Step `create PR` : `peter-evans/create-pull-request@<SHA-v6>` conditionnel sur `has_changes == 'true'`, avec `branch: chore/gdrive-sync-${{ github.run_id }}`, `title: chore(lore): gdrive sync <date>` (date via `date -u +%Y-%m-%d` step antérieur), `body:` inline shell+jq construit la ligne `Summary: N total, C created, ...` + fenced ```json bloc + checklist (AC-7), `labels: gdrive-sync, automation`, `draft: ${{ steps.summary.outputs.errors != '0' }}` (AC-8), `commit-message: chore(lore): gdrive sync run ${{ github.run_id }}`.
   k. Step `upload artifact` : `actions/upload-artifact@<SHA>` `name: gdrive-sync-summary.json` `path: /tmp/summary.json` `retention-days: 30` (AC-9), `if: always()` après run-sync.
4. **Runbook section** : appendre `## Onboarding gdrive-sync` après la section `## Sync Google Drive`. Sous-sections : (a) cross-ref scopes SA vers section existante, (b) `secrets.GDRIVE_SA_KEY` ajout via `gh secret set`, (c) `vars.GDRIVE_ROOT_FOLDER_ID` ajout via `gh variable set` (AC-15), (d) premier run `gh workflow run gdrive-sync.yml`, (e) review PR auto (AC-16).
5. **CHANGELOG** : `[Unreleased] / Added` → bullet `**ING-012** : workflow GitHub Actions \`gdrive-sync.yml\` (cron lundi 03:00 UTC + dispatch manuel) ...`.

## Risks / open questions
- **AC-5 entrypoint mismatch** : la spec écrit `uv run python scripts/gdrive_export.py --root-folder-id ...` mais `scripts/gdrive_export.py` n'existe pas — `scripts/gdrive_export/__main__.py` est la forme shipped (cf `cd scripts && uv run python -m gdrive_export ...` dans runbook ligne 226 + `__main__.py` doc). **Décision humaine** : utiliser la forme module `cd scripts && uv run python -m gdrive_export --root-folder-id "${{ vars.GDRIVE_ROOT_FOLDER_ID }}"`. Implementer note ce delta dans le commit message si reviewer le challenge.
- **PR body Summary line** : pas de modif script (Non-goals). Construction de la ligne `Summary: N total, ...` via inline shell `jq` step parsant `gdrive_sync.summary` event line depuis le stdout teed. Risk : format event peut évoluer ; verrouillé par ING-010 AC-17.
- **Working directory `cd scripts`** : la step `run sync` doit `working-directory: scripts` pour que `uv run` résolve le `pyproject.toml` local et que `__main__.py` localise `.gdrive_state.json` / `lore/` via paths relatifs (cf `_state_path`, `_lore_root`).
- **`peter-evans/create-pull-request` + `fetch-depth: 0`** : compatible mais doublonne le checkout interne ; ok.
- **Concurrency vs cron + dispatch coïncidence** : `cancel-in-progress: false` garantit série, mais un run queued peut dépasser le timeout 30 min global de l'autre. Acceptable phase 1.
