## Summary

PR-a (`ce76c9a feat(gateway): SEC-001 PR-a — JWT auth layer, fingerprint, /v1/me`) livre l'infrastructure auth lecture-seule : module `gateway/src/auth/{jwt,sessions,fingerprint,extractor}.rs`, middleware `resolve_identity` (JWT cookie/Bearer → session check DB → fallback fingerprint anonyme avec `Set-Cookie: archiviste_anon`), route `GET /v1/me` publique, propagation `X-User-Tier`/`X-User-Id` outbound chat, route test `cfg(debug_assertions)` `/v1/author-test`, migration `0003_users_auth` (colonnes nullable + sessions). Tests : 42+ verts (`me_test`, `auth_extractor_test`, `auth_properties` proptest INV-6, `chat_test` AC-14a). `cargo fmt --check`, `cargo clippy --all-targets -D warnings`, `cargo test` tous verts.

**Bloquant** : `fingerprint.rs:34` utilise `Uuid::NAMESPACE_DNS` au lieu de `Uuid::NIL` exigé verbatim par AC-10 ("namespace = NIL") — spec violation directe. Documentation interne contredit le code. Aucun test ne valide la valeur exacte du namespace, donc régression invisible.
