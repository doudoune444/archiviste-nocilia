# Plan — SEC-006 Gateway→workers ID-token signing + workers ingress=ALL

## Summary

Deliver the V1-deploy 502 fix as two coupled PRs: PR-A adds an `IdTokenProvider`
(sibling of `auth_metadata::TokenProvider`) that mints Google-signed ID tokens
via the Cloud Run metadata identity endpoint, wires it into `AppState`, and has
`forward_to_workers` attach `Authorization: Bearer <id-token>` on every outbound
`POST /v1/generate`. PR-B flips workers ingress to `INGRESS_TRAFFIC_ALL`,
bundles three already-staged worktree co-fixes (workers `container_port=8000`,
`LLM_PROVIDER`, `LLM_MODEL`), adds a Terraform `check {}` block guarding
against `allUsers`/`allAuthenticatedUsers` invoker bindings, ships a
`.tftest.hcl` fixture, and documents the post-deploy curl-403 smoke check.
IAM (`roles/run.invoker` restricted to `archiviste-runtime`) remains the sole
trust boundary; the ingress flip is routing, not auth.

## PR structure

- **PR-A — gateway (~245 LOC logical, target ≤300):** id_token module +
  AppState wiring + chat handler bearer attach + integration tests.
  Merges first; standalone (no infra dep). Branch:
  `feat/SEC-006-pr-a-id-token-signing`.
- **PR-B — infra + docs (~80 LOC):** ingress flip + bundled worktree co-fixes
  + Terraform check block + `.tftest.hcl` + runbook + CHANGELOG. Merges after
  PR-A. Branch: `feat/SEC-006-pr-b-workers-ingress-iam-check`.
- **Ordering rationale:** PR-A is harmless without PR-B (gateway just sends an
  ignored bearer to an internal-only workers). PR-B without PR-A would break
  `/v1/chat` immediately (workers IAM rejects unsigned calls). Hence A→B.

## Risks / open items

- **Z1 resolved** — single ctor `with_audience(audience: String)`. No
  `for_workers` sibling.
- **Z2 resolved** — `.tftest.hcl` ships in PR-B; CI wiring deferred (see
  follow-up below).
- **Z3 resolved** — reuse `classify_reqwest_error` from `token.rs` for
  `timeout`/`network`; HTTP non-2xx OR JWT body decode failure surfaces as
  `metadata_token_failed` via `TokenError::Fetch` (extend with a sibling
  variant only if the existing variant cannot carry the distinction).
- **Z4 resolved** — worktree disk state is the diff base for PR-B's pre-
  existing co-fix lines; no re-authoring.
- **Z5 resolved** — `tracing-test` `logs_contain` for AC-3 + AC-6 log
  assertions.
- **OQ-1 residual risk** — the `check` block hardcodes a reference to the
  one known `google_cloud_run_v2_service_iam_member.workers_runtime_invoker`
  resource. If a future PR adds a second IAM member resource keyed on
  workers (e.g., a new role binding), the assert silently passes on the new
  one. Mitigated by a WHY comment in `checks.tf` instructing the reviewer
  to extend the assert. Accepted residual.
- **OQ-2 residual risk** — `audience = config.workers_url` verbatim assumes
  Cloud Run `.uri` returns no trailing slash. Documented; unit test asserts
  the literal value. If a future provider version starts returning a
  trailing slash, the AC-6 integration test would still pass (mockito
  agnostic) but workers IAM would reject in prod with 401 → reported as
  `502/503` on `/v1/chat`. Acceptable failure mode (loud, logged).
- **OQ-5 resolved** — `WORKERS_URL` empty/malformed is already rejected by
  `Config::from_env` (`.context("WORKERS_URL env var required")?`) → boot
  fail. No new code.

  upstream failures stay opaque `503 upstream_unavailable`.
- **VPC connector** — spec non-goal; IAM is the trust boundary.
- **JWT signature verification** — spec non-goal; trust comes from the
  HTTPS connection to the metadata server, not the JWT.
- **Retry / exponential backoff on metadata fetch** — spec non-goal phase 1.
- **Cross-provider cache sharing** — spec non-goal (different audiences /
  semantics).
