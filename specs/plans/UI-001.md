# Plan — UI-001 Chat page MVP servie par la gateway (vanilla HTML+CSS+JS)

## Goal
Servir une page chat HTML+CSS+JS vanilla unique depuis la gateway Rust (`tower-http::ServeDir`) avec 4 en-têtes de sécurité posés sur tout le router, et un client JS qui fetch `POST /v1/chat`, gère `conversation_id` UUIDv4 en `localStorage`, et affiche user/assistant/error en `textContent`.

## Acceptance criteria recap
Voir `specs/acceptance/UI-001.md` AC-1..AC-19 (verbatim, non recopié — limite ≤ 100 lignes). OQ-1..OQ-5 résolus en spec.

- **Fichiers lus** : `specs/acceptance/UI-001.md`, `docs/adr/0005-frontend-vanilla-served-by-gateway.md`, `gateway/Cargo.toml`, `gateway/src/{lib,handlers/{mod,health,chat}}.rs`, `gateway/tests/chat_test.rs`, `specs/properties.md`, `specs/plans/GEN-002.md`, `infra/docker/gateway.Dockerfile`, `docker-compose.yml`, `CHANGELOG.md`. Glob `gateway/static/**` → vide (greenfield).
- **3 hypothèses** : (1) `tower-http` features `fs` + `set-header` = feature flip, pas d'ADR. (2) `ServeDir` couvre AC-5 path-traversal nativement (404 sur escape). (3) Header middleware au Router racine via `SetResponseHeaderLayer::if_not_present` × 4 couvre statiques + API + enveloppes erreur.
- **Incertitudes** : (Z1) CWD Docker = `/app` ; `COPY gateway/static /app/static` + chemin `static/` relatif. (Z2) `Content-Type` JS via `mime_guess` ∈ {`application/javascript`, `text/javascript`} — AC-2 tolère les deux. (Z3) AC-18 testé via absence d'event `chat` (TraceLayer toléré par spec). (Z4) `if_not_present` pour ne pas écraser headers handler existants.

## Risks / open questions
- **R1 (LOC)** : prod ≈ 40 (HTML) + 50 (CSS) + 110 (JS) + 20 (lib.rs) + 1 (Cargo.toml) + 2 (Dockerfile) = **~223 LOC prod**. Tests ~150 (hors comptage vertical-slice). OK ≤ 300.
- **R2 (CWD ServeDir)** : `cargo run` depuis `gateway/` voit `gateway/static/`. Docker runtime stage doit `WORKDIR /app` + `COPY static /app/static`. Si le binaire est exécuté depuis un autre CWD en prod, `ServeDir` 404. Mitigation : test d'intégration tourne aussi en CI depuis `gateway/`, runbook documente.
- **R3 (regex vs contains pour AC-7)** : `regex` n'est pas en dev-deps. Choix : `String::contains("<script src=")` etc + assertions négatives. Plus simple, suffisant phase 1. Pas d'ajout dep.
- **R4 (mime type AC-2)** : `mime_guess` peut renvoyer `application/javascript` ou `text/javascript`. AC-2 tolère les deux ; assert via `starts_with("application/javascript") || starts_with("text/javascript")`.
- **R5 (header `Content-Security-Policy` long)** : valeur littérale exacte AC-6 ; assertion `assert_eq!` byte-pour-byte sur la valeur entière, pas de parse-and-compare.
- **R6 (TraceLayer + AC-18)** : `TraceLayer` émet déjà des events. Spec tolère explicitement ("TraceLayer existante suffit"). Pas de test négatif sur TraceLayer ; juste s'assurer qu'aucun `tracing::info!` applicatif n'est ajouté pour les routes statiques (rien à coder = test trivialement vert).
- **R7 (Dockerfile)** : modifier `infra/docker/gateway.Dockerfile` côté runtime — actuellement pas de `WORKDIR` runtime, le binaire dans `/usr/local/bin/`. Ajouter `WORKDIR /app` + `COPY --from=builder /app/static /app/static`. Vérifier que les fichiers sont bien dans le contexte builder via la première `COPY`. Adapter ou ajouter `COPY static/ static/` au stage builder.
