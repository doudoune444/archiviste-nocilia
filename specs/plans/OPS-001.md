## Middleware design (D-H2)
- Custom `tower::Layer` wrap service handler chat. À l'entrée : capture `Instant::now()`. Appel inner service. Avant return : `total = start.elapsed()`. Soustraction depuis `Extension<WorkersCallDuration>` posé par `forward_to_workers` (chat handler insère `Extension(Duration)` après await reqwest). `overhead = total - workers_call`. Insert header `X-Gateway-Overhead-Ms: <u64 ms>` dans `response.headers_mut()`. Passthrough body intact.
- Si `Extension<WorkersCallDuration>` absent (ex. erreur validation 400) → overhead = total. Header posé quand même.

## Test strategy
- Integration Rust : `gateway/tests/overhead_header_test.rs` monte mock workers (wiremock) avec `Delay::new(200ms)`, envoie POST /v1/chat, assert (a) header `X-Gateway-Overhead-Ms` présent sur 200, (b) valeur entière, (c) `header < total_elapsed`, (d) `total_elapsed - header ≈ 200ms ± marge`.
- Test header présent aussi sur erreur 400 (validation fail).
- k6 dry-run manuel : `k6 run --env SCENARIO=chat_100_users --duration 5s` contre mock local, inspection `--http-debug=full`.
- Pas de property test (rien dans `specs/properties.md` ne couvre middleware timing).
- Pas de contract change (OpenAPI inchangé : header observational, pas spec).
