# Plan ŌĆö OBS-007 Runner OIDC auth for authenticated workers calls

## Goal
Attach a Google OIDC `Authorization: Bearer <id_token>` header to all runnerŌåÆworkers calls when the target is an https non-localhost URL, fail-fast (exit 5) if token fetch fails, no-op on local/CI ŌĆö code only, zero infra/contract change.

## Acceptance criteria recap
- AC-1: https + host Ōłē {localhost,127.0.0.1,::1} ŌåÆ every outbound workers call (`GET /healthz`, `POST /v1/retrieve`, `POST /v1/generate`) carries `Authorization: Bearer <id_token>` plus `X-Request-Id` (D-1).
- AC-2: `http://localhost:8000` (or host Ōłł {localhost,127.0.0.1,::1}, **or** scheme http) ŌåÆ no `Authorization`; headers stay `{"X-Request-Id": ...}` (D-1, no-op).
- AC-3: attach decision depends **only** on URL (scheme+host), no new CLI flag / env var / config option (D-1).
- AC-4: audience = `--workers-url` reduced to `scheme://host[:port]`; path/query/trailing slash stripped; port preserved (D-2).
- AC-5: token via injected `IdTokenProvider` seam (`audience: str -> SecretStr`); clients + `/healthz` probe receive it by injection; no real metadata call in tests (D-4).
- AC-6: authed target + provider fails ŌåÆ fail-fast **before any golden entry runs**, no retrieve/generate attempted, exit 5, no unauthenticated fallback (D-3).
- AC-7: token-fetch failure emits log `event=oidc_token_failed` with **no token bytes**, no raw credential (audience loggable) (D-3, ┬¦A09).
- AC-8: exit 5 distinct from {0,1,2,3,4}; header docstring documents `5 ŌĆö OIDC token fetch failure (authenticated target)` (D-3).
- AC-9: `eval/pyproject.toml` declares `google-auth>=2` in `dependencies` (core, not dev) + mypy `ignore_missing_imports` override for `google.auth`/`google.auth.*` (mirror psycopg2); no ADR (D-5).
- AC-10: non-authed target ŌåÆ `IdTokenProvider` never invoked (spy asserts zero calls) (D-1).
- AC-11: token carried as `pydantic.SecretStr` (redacts repr/str/log); no token bytes in run-file/stdout/stderr/logs, including failure path (┬¦A09, OBS-003 AC-10).
- AC-12: `specs/openapi/gateway-to-workers.yml` **not** modified ŌĆö transport concern, not contract (D-1).

## OQ resolutions
- **OQ-1 (module placement):** new `eval/oidc.py`. Justified: distinct responsibility (auth predicate + audience normalisation + provider seam) reused by both `clients.py` AND `ragas_runner.py`; inlining in `clients.py` would force `ragas_runner._check_workers_reachable` to import client internals and duplicate the predicate (3rd call site ŌåÆ extract, per clean-code DRY). Keeps `clients.py` a pure transport module.
- **OQ-3 (google-auth fetch fn):** `from google.oauth2.id_token import fetch_id_token` + `from google.auth.transport.requests import Request`; call `fetch_id_token(Request(), audience)`. Verified: on Cloud Run it sources the ID token from the metadata server automatically (no `GOOGLE_APPLICATION_CREDENTIALS` needed), returns a signed JWT string, raises on failure. No extra heavy dep (`google-auth` pulls `requests`/`cachetools`/`pyasn1`, < 1k LOC of new direct surface; no FFI) ŌĆö no ADR per D-5.

## Risks / open questions
- **Injecting a failing provider in subprocess CLI test (AC-6/7/11):** the runner builds its own `MetadataIdTokenProvider`; tests run via subprocess so monkeypatch won't reach it. Mitigation: `main()` accepts an optional injected provider param (default `None` ŌåÆ real provider), and the CLI test imports `main()` in-process with a fake provider that raises ŌĆö keeps AC-3 intact (no new CLI flag/env). Confirm this in-process invocation is acceptable vs. the existing subprocess `_run_runner` pattern.
- **`fetch_id_token` on non-GCP/localhost:** never called on non-authed path (AC-2/AC-10), so local dev/CI never hits metadata ŌĆö only the authed branch does, guarded by `is_authenticated_target`. If the lib API differs from verified signature at impl time ŌåÆ `no-workaround.md` blocker, stop.
- **`::1` host parsing:** `urllib.parse.urlsplit` yields bracketed `[::1]` in `.hostname` as `::1` (unbracketed) ŌĆö predicate must compare against `::1` unbracketed; audience reconstruction must re-bracket IPv6 in `netloc`. Edge case to cover in `test_oidc.py`.
- **`google-auth` transitive vuln surface:** `pip-audit` (CI A06 gate) must stay green; HIGH/CRITICAL on the new dep = blocker.
