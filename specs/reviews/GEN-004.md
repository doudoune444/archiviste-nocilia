## Security audit (.claude/rules/security.md)

- **A03 — SQL injection**: `ticket_service.py:84-126` uses asyncpg parameterized binding (`$1`, `$2`) throughout. The one f-string at line 80 (`SET LOCAL statement_timeout`) interpolates an `int()` cast of a `Final` module constant — no user input. **OK**.
- **A04 — DB timeout**: `TICKET_SQL_TIMEOUT_SECONDS = 5.0` enforced via `SET LOCAL statement_timeout` inside the transaction. **OK**.
- **A09 — Logging**: ALERT logs `ticket_service_failed` emit only `request_id`, `reason`, `error_type` — never the raw query nor the embedding vector. **OK**.
- **RAG prompt injection**: `build_lore_gap_messages` injects user query into the **HumanMessage** role only, with the `[user query, suspected injection]: ` prefix when applicable. **No chunks** are passed into the LLM lore_gap call (closes W-I-1 by construction). **OK**.
- **Embedding poisoning**: lore-gap tickets store the raw query in `tickets.question` (AC-12 explicit) but the question is NOT routed back into any prompt — only stored for the author dashboard. UI-002 will handle HTML-escape on read (out of scope per spec). **OK**.
- **Secrets**: no API keys, tokens, or credentials in the diff. **OK**.
- **Trust boundary**: `user_tier` continues to flow from gateway headers (SEC-001 already handles this); GEN-004 does not introduce a new trust boundary. **OK**.
- **Output sanitization**: LLM lore-gap output is returned as plain text in `answer` field, identical to canon path. Web rendering layer (out of scope) must HTML-escape. **OK**.
