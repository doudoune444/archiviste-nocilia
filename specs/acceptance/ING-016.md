# ING-016 — Chunker tokenizer swap (BGE-M3 → Mistral)

## Status

draft

## Context

INFRA-002 PR-d a swappé l'embedder runtime BGE-M3 → `mistral-embed` mais a dû garder `transformers>=4.45` en dépendance runtime parce que `workers/src/archiviste_workers/ingest/chunker.py:8` importe `AutoTokenizer` et appelle `AutoTokenizer.from_pretrained("BAAI/bge-m3")` pour construire le LangChain text splitter. Voir `docs/blockers.md` entrée 2026-05-18 — INFRA-002.

Image workers runtime conserve donc ~100 MB pour `transformers` (sans `torch`) au lieu de droper complètement la dépendance HF. `sentence-transformers` (~2 GiB de poids modèle) a été drop ; gain d'image atteint pour INFRA-002 mais ce ticket finit le travail.

## Acceptance criteria

- AC-1 : `workers/src/archiviste_workers/ingest/chunker.py` n'importe plus `transformers`
- AC-2 : Splitter utilise tokenizer Mistral (via `mistralai` SDK ou `tiktoken` cl100k_base fallback documenté) OU splitter character-based avec mapping documenté équivalent BGE-M3 chunks
- AC-3 : Test unitaire `tests/test_chunker.py` vérifie que chunks de référence (golden_qa.jsonl ingest sample) restent dans tolérance de ±15 % LOC vs ancien split BGE-M3
- AC-4 : `workers/pyproject.toml` `[project.dependencies]` ne contient plus `transformers` (drop final)
- AC-5 : `workers/pyproject.toml` `[project.optional-dependencies] embedder-fallback` documente `transformers + sentence-transformers` ensemble (V2 self-host option)
- AC-6 : Image workers runtime perd ≥ 80 MB (mesure `docker images archiviste-workers --format '{{.Size}}'` avant/après)
- AC-7 : `eval/ragas_runner.py` sur golden_qa.jsonl reste ≥ baseline INFRA-002 sur faithfulness + context_precision

## Non-goals

- Pas de re-ingestion complète canon V1 (re-chunk uniquement si nouvelle ingestion)
- Pas de migration vers chunker LangChain `RecursiveCharacterTextSplitter` autre que pour fallback documenté
- Pas de tuning chunk_size (rester sur valeurs actuelles)
