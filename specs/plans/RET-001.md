## Goal

Workers expose `POST /v1/retrieve` that embeds a query with bge-m3, runs a single ACL-filtered cosine top-K SQL against `chunks` via the existing asyncpg pool, returns ordered chunks with deterministic tie-break, and emits one redacted JSON log per request.

## Acceptance criteria recap

- AC-1 : Le worker FastAPI expose `POST /v1/retrieve` accessible sur le port workers (8000) ; un appel valide retourne `200` avec `Content-Type: application/json; charset=utf-8`.
- AC-2 : Le body de requête est un JSON de schéma `{"query": str, "top_k": int|absent, "user_tier": str}` ; tout autre champ supplémentaire est ignoré silencieusement, toute clé requise manquante retourne `400 {"error":"invalid_request"}`.
- AC-3 : `query` doit être une chaîne non-vide après strip et de taille ≤ 4096 octets UTF-8 ; sinon `400 {"error":"invalid_query"}`.
- AC-4 : `top_k` est un entier optionnel ; absent ou `null` → valeur par défaut `5` ; présent et hors `[1, 20]` (bornes incluses) → `400 {"error":"invalid_top_k"}`.
- AC-5 : `user_tier` est obligatoire et doit appartenir exactement à l'ensemble `{"anonymous","members","author_only"}` ; absent ou hors ensemble → `400 {"error":"invalid_user_tier"}`.
- AC-6 : La query est embeddée via le wrapper bge-m3 réutilisé de ING-001 (modèle `BAAI/bge-m3`, dimension 1024) ; le modèle est chargé une fois au démarrage du worker (lifespan startup), aucun chargement n'a lieu pendant le traitement d'une requête.
- AC-7 : La recherche exécute une requête SQL paramétrée unique utilisant l'opérateur pgvector `<=>` (cosine distance) sur `chunks.embedding`, joignant `documents` pour appliquer le filtre ACL post-retrieval `WHERE documents.access_tier = ANY($allowed_tiers)` et `LIMIT $top_k`, où `$allowed_tiers` est dérivé de `user_tier` selon la table : `anonymous → {public}`, `members → {public, members}`, `author_only → {public, members, author_only}`.
- AC-8 : Le filtre ACL est exprimé dans la clause SQL `WHERE` (pas en post-filtrage applicatif) ; un test d'intégration insérant 1 chunk `public` + 1 `members` + 1 `author_only` puis appelant `user_tier="anonymous"` retourne uniquement le chunk `public`, jamais les autres, indépendamment des scores cosine.
- AC-9 : La réponse `200` a la forme exacte `{"chunks":[{"chunk_id":str,"document_id":str,"source_path":str,"ord":int,"text":str,"score":float}, ...], "embedding_ms":int, "search_ms":int}` ; `chunk_id` et `document_id` sont des UUID au format canonique RFC 4122 ; `source_path` provient de `documents.source_path`.
- AC-10 : `score` est calculé comme `1 - (chunks.embedding <=> $query_embedding)` (similarité cosine ∈ `[0.0, 1.0]`) et arrondi à 6 décimales ; le tableau `chunks` est ordonné par `score` décroissant (égalités résolues par `chunk_id` ASC pour déterminisme).
- AC-11 : `len(chunks) ≤ top_k` toujours ; aucun doublon `chunk_id` dans la réponse.
- AC-12 : Si la recherche ne renvoie aucune ligne, la réponse est `200 {"chunks":[], "embedding_ms":N, "search_ms":M}` — jamais `404` ni `204`.
- AC-13 : Embedder non chargé → `503 {"error":"embedder_unavailable"}`, requête n'atteint pas la DB.
- AC-14 : Connexion PostgreSQL en échec → `503 {"error":"database_unavailable"}` ; aucune trace SQL/erreur DB ne fuit.
- AC-15 : `embedding_ms` et `search_ms` mesurés isolément, entiers `≥ 0`.
- AC-16 : Log JSON unique `{"event":"retrieve","query_len":N,"top_k":K,"user_tier":T,"results":R,"embedding_ms":E,"search_ms":S,"status":"ok|invalid_query|invalid_top_k|invalid_user_tier|embedder_unavailable|database_unavailable"}` ; jamais `query`, `text`, `embedding`, ni message DB.
- AC-17 : `specs/openapi/gateway-to-workers.yml` mis à jour ; `schemathesis run` passe sur `/v1/retrieve`.

   SELECT c.id AS chunk_id,
          d.id AS document_id,

## Risks / open questions

- **Real bge-m3 in CI**: model weights ~2.3 GB. ING-001 already faces this; reuse whatever caching strategy ING-001 settled on (pre-pulled in CI image or `HF_HOME` cache). If absent, surface as blocker — do **not** add a fake embedder fallback.
- **`asyncio.TimeoutError` vs `TimeoutError`**: on Python 3.12 `asyncio.TimeoutError = TimeoutError`. Catch both explicitly to keep mypy strict happy.
- **`Content-Type: application/json; charset=utf-8`** (AC-1): FastAPI's `JSONResponse` default media type is `application/json` (no charset). Need to confirm via integration assert ; if FastAPI strips the charset, return a custom `Response(content=..., media_type="application/json; charset=utf-8")`. Decided at step 4 when writing the failing test.
- **Schemathesis vs `/v1/generate` stub**: the existing `/v1/generate` schema is unimplemented; running schemathesis without a path filter would fail unrelated to RET-001. Plan restricts the run to `/v1/retrieve`. Out-of-scope cleanup of `/v1/generate` is a future ticket.
- **Pending humain-only edits**: `specs/properties.md` INV-9/INV-10 must be merged by humain in the same PR window for the property test references to be valid. Plan flags this as a blocker if the humain hasn't approved before merge.
- **embedding NaN edge-case**: bge-m3 with `normalize_embeddings=True` already rules out NaN unless input is empty (rejected at validation). No additional guard.
