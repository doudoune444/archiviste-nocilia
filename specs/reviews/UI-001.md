## Security

- **No secrets** in `gateway/static/*` (verified — no API key, no token, no JWT, no DB connection string).
- **CSP** posed router-wide (literal exact value matches AC-6). `script-src 'self'` + `style-src 'self'` + `frame-ancestors 'none'` + `base-uri 'none'` + `object-src 'none'` + `form-action 'self'` covers A05.
- **XSS** : defense in depth — (a) CSP, (b) `textContent`-only DOM injection at app.js:35,40 ; no raw-HTML sinks anywhere in the static bundle.
- **Path traversal (A01/A05)** : `ServeDir`/`ServeFile` natively reject `..` and `%2e%2e` (tests prove this). `nest_service("/assets", ServeDir::new("static/assets"))` constrains to that subdir.
- **Trust boundary** : client never injects `user_tier` / `user_id` (hardcoded gateway-side per GEN-002 AC-3). The body sent to workers is `{query, conversation_id}` only — sentinel injected by gateway handler.
- **Same-origin fetch** with `credentials: 'same-origin'` — no CSRF surface (no cookies set anywhere).
- **No SSRF** (no user-supplied URL flows into a fetch).
- **No SQL injection** surface (no DB writes added).
- **Rate-limit absent** on static routes — explicitly out-of-scope per spec §Non-goals ("ticket SEC-* dédié couvrira l'ensemble du gateway").
- **HSTS absent** — explicitly out-of-scope per spec §Non-goals (TLS termination upstream Cloud Run / LB).
- **`X-Request-Id` exposure** via `<small>req: <id></small>` is by design (AC-14). UUIDv4 is non-sensitive (no user ID/email leakage).
- **CORS** : not configured here (same-origin only). No wildcard + credentials risk.
